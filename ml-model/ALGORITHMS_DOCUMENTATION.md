# Machine Learning Algorithms Documentation

This document provides a detailed overview of the Machine Learning algorithms and methodologies used in the LoanHarmony Nexus project for credit risk evaluation and loan approval prediction.

## Overview

The system utilizes an **Ensemble Learning** approach, where multiple independent models analyze borrower data to provide a consolidated risk assessment. This methodology ensures higher accuracy and robustness by reducing the impact of individual model biases.

---

## 1. CatBoost (Categorical Boosting)
**Role:** Primary Decision Engine

CatBoost is a high-performance open-source library for gradient boosting on decision trees. It is the lead model in our system due to its superior handling of categorical data without extensive preprocessing.

- **Key Features:**
  - Optimized for handling categorical features (e.g., Employment Status, Loan Purpose).
  - Robust against overfitting through symmetric tree structures.
  - Handles missing values automatically.
- **Project Usage:** Provides the final decision threshold used by the system for the "APPROVE/REVIEW/REJECT" status.

## 2. XGBoost (Extreme Gradient Boosting)
**Role:** Complementary Gradient Boosting

XGBoost is an optimized distributed gradient boosting library designed to be highly efficient, flexible, and portable.

- **Key Features:**
  - Implements machine learning algorithms under the Gradient Boosting framework.
  - Highly scalable and supports parallel processing.
  - Includes built-in regularization (L1/L2) to prevent overfitting.
- **Project Usage:** Acts as a validation model to cross-verify the probabilities generated by CatBoost.

## 3. Random Forest
**Role:** Bagging Classifier

Random Forest is a popular ensemble method that operates by constructing a multitude of decision trees at training time and outputting the class that is the mode of the classes.

- **Key Features:**
  - Excellent at handling non-linear relationships in data.
  - Provides "Feature Importance" metrics, helping us understand which borrower traits matter most.
  - Less sensitive to outliers in numerical data (like income or debt ratio).
- **Project Usage:** Used to ensure the model remains stable across varied borrower demographics.

## 4. Logistic Regression
**Role:** Baseline Statistical Model

Logistic Regression is a fundamental statistical model that uses a logistic function to model a binary dependent variable.

- **Key Features:**
  - Highly interpretable and mathematically simple.
  - Provides a linear baseline to ensure complex ensemble models are actually providing significant gain.
  - Fast execution time for real-time predictions.
- **Project Usage:** Serves as the "sanity check" model. If all complex models diverge significantly from the logistic baseline, the system flags the application for manual review.

## 5. Supporting Assets and Preprocessing Components

To ensure the algorithms perform accurately, the system relies on several supporting assets that handle data transformation and metadata management.

### A. One-Hot Encoder (`encoder.pkl`)
- **Purpose:** Converts categorical text data into a format that machine learning models can understand.
- **Functionality:** It maps labels (e.g., "Salaried", "Self-Employed") into binary vectors. This ensures that the models do not assume an alphabetical or numerical order where none exists.

### B. Standard Scaler (`scaler.pkl`)
- **Purpose:** Normalizes numerical data.
- **Functionality:** Features like "Monthly Income" (in thousands) and "Credit Score" (in hundreds) have very different scales. The Scaler standardizes these so that no single feature dominates the model's decision simply due to its magnitude.

### C. Metadata Repository (`metadata.pkl`)
- **Purpose:** Stores the "DNA" of the training dataset.
- **Functionality:** It keeps track of the exact feature names, the order of columns, and the training parameters. This ensures that the production environment (`app.py`) processes user data in the exact same way the models were originally trained.

### D. Consolidated Assets (`all_loan_prediction_assets.pkl`)
- **Purpose:** All-in-one deployment package.
- **Functionality:** Combines all models (CatBoost, XGBoost, etc.) and the supporting assets (Encoder, Scaler) into a single serialized file for fast loading and low-latency predictions.

## Technical Methodology

### Data Preprocessing
- **Categorical Encoding:** One-Hot Encoding via `encoder.pkl`.
- **Feature Scaling:** StandardScaler via `scaler.pkl`.
- **Age Calculation:** The system automatically converts `Date_of_Birth` into numerical `Age` for better model ingestion.

### Prediction Logic
Each model outputs an **Approval Probability (0.0 to 1.0)**.
- **Threshold:** A standard threshold of **0.6** is used.
- **Decision Matrix:**
  - Probability > 0.6: **APPROVE**
  - Probability 0.4 - 0.6: **REVIEW**
  - Probability < 0.4: **REJECT**
